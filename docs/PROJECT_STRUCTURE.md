# Project Structure

This folder is a ROS 2 workspace plus offline pipeline tools. Keep the top-level
layout stable because ROS 2 expects `src/`, `build/`, `install/`, and `log/` to
live at the workspace root.

## Source And Inputs

`data/`
: Canonical demo inputs and outputs for the planner pipeline.
  `demoN_map.json` is the map input. `demoN_case.json` is the executable case
  consumed by ROS, RViz, and Gazebo exporters.

`src/swarm_random_payload/`
: The ROS 2 Python package. It contains launch files, RViz config, mesh assets,
  package-local copies of case data, and runtime Python nodes/models.

`tools/`
: Offline Python tools. These generate cases, verify cases, export Gazebo
  trajectories/worlds, freeze a uniform recording trajectory, and export pose
  streams for the Gazebo replay plugin.

`scripts/`
: User-facing command wrappers. Prefer these over long raw commands when
  running demos.

`figures/`
: Map-design helper UI and Drake IRIS server. This is for authoring/editing
  maps, not for running the final demo.

`gazebo_pose_replay/`
: C++ Gazebo system plugin source. The recording demo builds this plugin and
  uses it to replay all model poses in-process, avoiding slow per-frame service
  calls.

## Generated Artifacts

`out/`
: Generated Gazebo worlds, cached trajectories, pose streams, and saved
  snapshots. It is useful locally for recording and rollback, but it is not the
  conceptual source of the pipeline.

`screenshots/`
: Captured screenshots for visual checks.

`build/`, `install/`, `log/`
: Generated by `colcon build`. Do not manually edit them. If they become
  inconsistent, rebuild the workspace.

`.python_drake/`
: Local Drake Python dependency bundle used for IRIS generation on this
  machine. It is machine-local and should not be treated as paper/source logic.

## Runtime Flow

```text
data/demoN_case.json
  -> src/swarm_random_payload/swarm_random_payload/random_payload_model.py
  -> src/swarm_random_payload/swarm_random_payload/paper_qp_controller.py
  -> src/swarm_random_payload/swarm_random_payload/random_payload_node.py
  -> RViz markers / poses / paths
```

Gazebo recording flow:

```text
data/demoN_case.json
  -> tools/export_gazebo_trajectory.py
  -> tools/freeze_uniform_recording_trajectory.py
  -> tools/export_gazebo_scene.py
  -> tools/export_gazebo_pose_stream.py
  -> gazebo_pose_replay/PayloadPoseReplaySystem.cc
  -> Ignition Gazebo GUI
```
