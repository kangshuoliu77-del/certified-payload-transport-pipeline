# File Guide

This is the quick file map for the current standard folder.

## Top-Level

`README.md`
: Main entry point and recommended commands.

`docs/PROJECT_STRUCTURE.md`
: Detailed folder structure and source/generated boundary.

`docs/RUNBOOK.md`
: Practical commands for build, RViz, Gazebo, rebuild, and verify.

`docs/CODE_READING_ORDER.md`
: The order to read the code without getting lost in generated files.

`DEMO_INDEX.md`
: Fixed numbering and role of demo1 through demo8.

`CINEMATIC_DEMO.md`
: Older cinematic notes kept for reference.

## Canonical Data

`data/case_manifest.json`
: Rebuild manifest for the standard cases.

`data/demoN_map.json`
: Map input: obstacles, task points, safety distances, formation scale, and
  seed settings.

`data/demoN_case.json`
: Executable case consumed by RViz, Gazebo exporters, and verification tools.

## Core Pipeline Tools

`tools/generate_case.py`
: Core case generator. This is where IRIS regions, formation feasibility,
  bridge certificates, route states, and transitions are built.

`tools/rebuild_cases.py`
: Rebuild one or more demos from the manifest.

`tools/verify_cases.py`
: Verify generated cases and paper-QP execution checks.

## ROS/RViz Package

`src/swarm_random_payload/package.xml`
: ROS package metadata.

`src/swarm_random_payload/setup.py`
: Installs Python modules, launch files, config, meshes, and package data.

`src/swarm_random_payload/launch/case_payload_demo.launch.py`
: Main RViz/ROS launch file for a case.

`src/swarm_random_payload/launch/cinematic_payload_demo.launch.py`
: Presentation RViz launch file.

`src/swarm_random_payload/config/cinematic_payload_demo.rviz`
: RViz display configuration.

`src/swarm_random_payload/swarm_random_payload/random_payload_node.py`
: ROS node that publishes markers, poses, paths, regions, bridge highlights,
  drones, and payload.

`src/swarm_random_payload/swarm_random_payload/random_payload_model.py`
: Runtime scenario model and validation checks.

`src/swarm_random_payload/swarm_random_payload/paper_qp_controller.py`
: Paper-style QP controller.

## Gazebo Recording Layer

`tools/export_gazebo_scene.py`
: Export an SDF world from a case file.

`tools/export_gazebo_trajectory.py`
: Export paper-QP execution frames for Gazebo.

`tools/freeze_uniform_recording_trajectory.py`
: Convert execution frames into a smoother uniform recording trajectory.

`tools/export_gazebo_pose_stream.py`
: Export per-frame model poses for the replay plugin, including drones,
  payload, ropes, camera target, current/target regions, bridge highlights, and
  cable timing.

`gazebo_pose_replay/PayloadPoseReplaySystem.cc`
: C++ Gazebo replay plugin.

`scripts/run_gazebo_recording_lift_demo.sh`
: Current polished Gazebo video demo.

## Generated Or Local-Only

`build/`, `install/`, `log/`
: Generated by colcon.

`out/`
: Generated Gazebo worlds, trajectories, pose streams, and local snapshots.

`screenshots/`
: Visual check screenshots.

`.python_drake/`
: Local Drake dependency bundle for IRIS generation.
